## MySQL核心配置

| 参数名                         | 参数值         | 描述                                                         |
| :----------------------------- | :------------- | ------------------------------------------------------------ |
| innodb_flush_log_at_trx_commit | 1              | redo log 的写盘策略：  <br />`0`：每秒刷新一次日志缓冲区到日志文件，事务提交时不主动触发刷新。 <br />`1`（默认）：每次事务提交时都会刷新日志缓冲区到日志文件，确保事务的持久性。 <br />`2`：每次事务提交时将日志缓冲区内容写入到操作系统缓存，但每秒由操作系统决定是否刷新到磁盘。 |
| innodb_thread_concurrency      | 128            | 限制InnoDB可以同时执行的线程数量：<br /> 0（默认）：不限制并发线程的数量。 <br />大于0的值：限制并发线程的数量，超过该值的线程将等待或睡眠一段时间后再次尝试。 |
| max_allowed_packet             | 33554432       | 定义客户端和服务器之间传输的最大数据包大小（单位：B）：<br />通常设置为几兆字节（例如 `16M` 或 `32M`），根据应用需求和网络条件调整 |
| max_connections                | 10000          | 设置MySQL服务器可以同时维持的最大连接数：<br />根据服务器的负载能力和资源限制来设置 |
| table_open_cache               | 6000           | 控制MySQL服务器为了快速打开表而维护的打开表的缓存数量：<br />根据服务器的内存大小和并发请求的数量来设置 |
| transaction_isolation          | READ-COMMITTED | 事务的隔离级别                                               |
| long_query_time                | 1.000000       | 定义MySQL认为查询执行时间长，需要记录日志的阈值（以秒为单位）<br />例如设置为 `10` 表示执行时间超过10秒的查询将被记录 |
| sync_binlog                    | 1              | 控制MySQL二进制日志（binlog）的写入策略<br />`0`：每次事务提交后，binlog缓存中的事件将写入到文件系统缓存，由操作系统决定何时刷写到磁盘。 <br />`1`（默认）：每次事务提交后，binlog缓存中的事件将被写入到磁盘。<br /> `N`（`N`大于1）：每`N`个事务提交后，binlog缓存中的事件将被写入到磁盘。 |
| innodb_lock_wait_timeout       | 50             | 设置InnoDB事务等待锁定资源的最大时间（以秒为单位）<br />例如设置为 `50` 表示事务在等待锁定资源超过50秒后将被回滚。 |
